# Ejercicios prácticos

Practica lo que has aprendido con tareas reales. Cada ejercicio está ligado a un módulo específico — aprendes la funcionalidad, luego la aplicas con datos reales.

## Cómo funciona

Cada ejercicio tiene **dos partes**:
1. **La tarea** — un reto de negocio realista con archivos de datos reales
2. **La funcionalidad** — la resuelves construyendo algo con el módulo que acabas de aprender (un slash command, una skill, un subagente, un hook, etc.)

No solo usas Claude Code — lo extiendes.

## ¿Para quién son?

Para todos. Las tareas son universales — lo que cambia es **qué construyes** con cada funcionalidad de Claude Code.

| Rol | Ejercicios que te resultarán familiares |
|-----|------------------------------------------|
| **Ventas** | Limpieza de CRM (#1), investigación de mercado (#2), análisis de conversaciones (#4) |
| **Finanzas** | Categorización de gastos (#1), análisis de datos (#4), taxonomía (#3) |
| **RRHH / Personas** | Estandarización de taxonomía (#3), análisis de conversaciones (#4) |
| **Operaciones** | Evaluación de proveedores (#2), integración de sistemas (#7) |
| **Producto** | Evaluación de IA (#5), pipeline de contenido (#6) |
| **Marketing** | Investigación (#2), QA de contenido IA (#5), contenido end-to-end (#6) |

## Ruta recomendada

- **Después de Empezar (00-intro → 02-memory):** Ejercicios 1, 2, 3
- **Después de Funcionalidades Core (03-skills → 05-mcp):** Ejercicios 4, 5
- **Después de Avanzado (07-plugins, 08-checkpoints):** Ejercicios 6, 7

## Consejos para tener éxito

| Hacer | Evitar |
|-------|--------|
| Completa el módulo antes del ejercicio | Saltar al ejercicio sin aprender la funcionalidad |
| Construye el artefacto reutilizable (comando, skill, agente) | Solo completar la tarea sin la funcionalidad |
| Prueba tu artefacto con entradas distintas | Hardcodearlo solo para los datos de este ejercicio |
| Lee la tabla "Conexión con el módulo" en cada ejercicio | Ignorar qué concepto de la funcionalidad practicar |
