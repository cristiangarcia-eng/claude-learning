# Why Claude Code?
**One command. No coding. Your first real win in under 10 minutes.**

Last Thursday, Marta — head of ops at a 40-person logistics company — had
a problem. 287 supplier invoices sitting as PDFs across 4 email threads,
due in her CFO's inbox as a clean Excel by 9am Friday. At 5pm she opened
Claude Code, dragged the folder into the terminal, and said:

> *"Read every PDF, extract the invoice number, vendor, amount and due
> date, and put it in a spreadsheet sorted by due date."*

At 5:03pm she had the Excel.

Marta had never written a line of code in her life.

---

> **Don't let the name fool you.** It's called "Claude Code," but you
> don't need to write a single line of code to use it. What you actually
> have is a full AI agent with access to your computer — it reads
> documents, browses the web, controls apps, and executes multi-step
> tasks autonomously. Market research, competitive analysis, data
> cleanup, content pipelines, report generation — all of that is Claude
> Code territory. The "Code" in the name refers to how it works under
> the hood, not what you need to know.

## The 3-second version: what makes it different

You've used ChatGPT. You type, it answers, you copy-paste into Word or
Excel, and then you do the actual work yourself. Claude Code flips that:

| Chat AI (what you know) | Claude Code (what you'll learn) |
|---|---|
| **You** do the work. It gives advice. | **Claude** does the work. You describe the outcome. |
| It sees only what you paste. | It reads **every file** in your project. |
| You manually apply every change. | It edits, creates, and organizes files for you. |
| One question at a time. | Multi-step tasks, end to end. |

Regular AI chat is texting a smart friend for advice. Claude Code is
that friend sitting down at your computer and doing the work with you —
for 4 hours straight if you ask it to.

## 5 places to run Claude Code

Same AI everywhere. You just pick where it lives:

| Where | Best for | The trade-off |
|---|---|---|
| **Terminal (CLI)** | Max power, zero limits | ~10 min of terminal basics |
| **Cursor** | Code + conversation side by side | Same power as terminal |
| **Desktop App** | Visual diffs, scheduled jobs | Prettier UI, slightly less flexible |
| **Web (claude.ai/code)** | Quick tasks, nothing to install | Runs in the cloud, not your laptop |
| **Mobile (iOS)** | On the go | Tied to a web session |

**In this course we use Cursor** — because you'll see Claude edit files
in real time, right next to the conversation. The skills transfer 1-to-1
to every other surface, so you're not locking yourself in.

## How you'll start (in 60 seconds)

1. Open Cursor
2. Hit `` ` `` to open the terminal panel at the bottom
3. Type `claude` and press enter
4. Start talking in plain English

That's it. **One command. No syntax. No setup rituals.** From here on,
you describe what you want and Claude figures out how.

## What this course turns you into

This course takes you from "never opened a terminal" to "shipping real
work with Claude" in a weekend. Every lesson is designed for people who
don't write code — product managers, designers, sales leads, ops,
founders, and the plain curious.

By the end, you'll be able to:

- **Set up Cursor** as a workspace that feels like yours *(15 min, one time)*
- **Talk to Claude with your voice** instead of typing *(huge for long prompts)*
- **Have Claude clean, analyze, and organize your files** *(the 40-min-per-week kind of task)*
- **Connect Claude to your real tools** — Gmail, Notion, Slack, your CRM
- **Automate the 5 things you do every week** that you secretly hate doing

No code. No prior AI experience. If you can describe what you want in
plain English, **you'll be using Claude Code by the end of this lesson —
not the end of this course.**
