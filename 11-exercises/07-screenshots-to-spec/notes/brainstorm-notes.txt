FOCUS BLOCKS - Brainstorm Notes
================================
Meeting: March 15 product brainstorm
Attendees: Jamie (PM), Riley (Design), Morgan (Eng Lead), Casey (CS)

CORE IDEA
- Users can schedule "focus blocks" on their task list
- During a focus block, notifications are snoozed automatically
- Tasks assigned to a focus block get promoted to a special view
- Should feel like "do not disturb" but smarter

WHAT CASEY SAID (from customer feedback)
- "Our power users say they get interrupted 15-20 times/day by our own notifications"
- "Three enterprise accounts specifically asked for DND mode in their last QBR"
- Customers want to block 2-3 hour windows for deep work
- Some want recurring focus blocks (every morning 9-11am)
- Request from Acme Corp: "let me pick which notification types still come through"

RILEY'S DESIGN THOUGHTS
- Could live in the calendar sidebar
- Or maybe a floating button/toggle?
- Need to think about mobile vs desktop -- mobile might need different UX
- "What happens when a focus block ends? Do all the snoozed notifs flood in?"
- Suggested a gentle "re-entry" experience -- summary of what you missed
- Color coding? Like a purple bar across the time block?
- Worry: if it is too hidden nobody will use it

MORGAN'S TECHNICAL NOTES
- Notification system is already event-based so snoozing is doable
- Need to handle: what if someone @mentions you in urgent?
- "Urgent override" -- certain notification types bypass the block
- Calendar integration would be nice but adds 3-4 weeks
- MVP could be manual toggle + scheduled blocks
- Phase 2: auto-detect from calendar (if user has a meeting-free block)
- Storage: just need a new table for focus_blocks with start/end times
- Websocket implications -- need to suppress real-time updates too

QUESTIONS NOBODY ANSWERED
- Do focus blocks apply per-project or globally?
- What about shared/team focus blocks? (like "team deep work hour")
- How does this interact with our existing "snooze" feature on individual tasks?
- Pricing: is this a free feature or premium?
- What happens to deadlines/reminders during a focus block?

USER PERSONAS WE DISCUSSED
- Solo freelancer using Orbit for personal task management
- PM at a mid-size company managing 5-6 projects
- Engineering manager who wants to protect their team's focus time
- Executive who wants a daily "strategic thinking" block

POSSIBLE SUCCESS METRICS (rough)
- Adoption rate (how many users try it)
- Retention of focus block users vs non-users
- "Productivity score" somehow? Hard to measure
- NPS change?
- Reduction in "notification fatigue" complaints to CS

COMPETITIVE LANDSCAPE (quick notes)
- Notion has a "focus mode" but it is just hiding the sidebar
- Todoist has nothing like this
- Sunsama has time-blocking but no notification control
- This could be a real differentiator

RISKS
- Overengineering it -- MVP should be simple
- Users might set focus blocks and forget, missing critical stuff
- If the "urgent override" is too easy to trigger it defeats the purpose
